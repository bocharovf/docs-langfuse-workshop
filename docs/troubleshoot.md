# Если возникли проблемы

## Проект не запускается

Попробуйте удалить все контейнеры и volume с именем langfuse*.

```
docker compose down

docker volume ls

docker volume rm langfuse-workshop_qdrant_data

docker volume rm ...

...

docker compose up -d
```

Проверьте настройки в `.env`, они должны выглядеть примерно так:

![Пример содержимого .env](/img/env-example.png)

## Агент выдает ошибку при обработке запроса

Убедитесь, что все контейнеры запущены с помощью `docker ps -a`, должно быть так:

![Пример содержимого .env](/img/docker-ps.png)

Убедитесь, что в <https://console.mws.ru/> вы корректно создали сервисный аккаунт и развернули все 3 модели (как описано в разделе Подготовка).

## Сценарий не воспроизводится

Убедитесь, что выставлены корректные флаги в `.env` и изменения сохранены.

Выполните полный перезапуск проекта 

```
docker compose up -d --force-recreate
```
